Classification of Poisonous vs. Edible Mushrooms Using Machine Learning/Decision Trees

How to use a mushroom dataset as a guide in detecting poisonous vs. edible. This data is available via UCI, but I've put the csv on my github for ease of access.

10-01-2021

The goal of this was to show how to implement a decision tree model to something practical. After a couple of years studying satistics and taking courses I wanted do something with practical application; no matter how small they might be. The dataset and scripts are available on my github.

There are a serveral models you can use to predict outcomes - but I wanted something that had tracable steps and that could be used as a guide. Many models can input a value, but not show the steps on how they got there; decision trees are different - you visually see the steps that model has taken to get to it's outcome.

How I Made this Model

I originally found this dataset in my Applied Machine Learning Course that I took through the University of Michigan. From this dataset we were instructured to find the Gini importance of each features, and then with another unrelated dataset asked to make a decision tree. After determining the importance of each feature, I decided to see how I could transform this into a simple to follow decision tree that could potentially be used to classify edible vs. poisonous mushrooms.

To do this I used Jypyter notebooks, and put all of the data etc. that I'd be working on in a local directory to work on. The steps I followed today are presented in a cleaner order than the steps it took to put this together - I had attempted to input the data into the classifier without doing data-cleanup. This cleanup didn't impact the outcomes but it lead to a dataset/tree that was unreadable. Most of the time I spent in this project was to gain more understanding on how decision trees worked, what a random forest was, what Gini importnace is, and reading documentation on visualizing the tree - then using graphviz to manually populate the tree, and re-labelling the fields or features of the model.

The Outcome

I wanted to produce something that was useful (with some element of risk) to forage (collect) mushrooms. Below is the outcome of this project, using this model the user would be able to look at a mushroom and answer a question, if the answer to that question was true, you would move left on the diagram; if false you move to the right. If you end in a coral coloured node (leaf) it means it is highly likely the mushroom is poisonous, if blue it is edible. As stated, this isn't a flawless model despite the metrics/accuracy score; you need to know the risks associated with this activity, and consult and expert.
