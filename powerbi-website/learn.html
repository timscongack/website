What is PowerBI?

PowerBI is a Microsoft application that allows users to load and transform datasets in a programmatic way and then visualize and display data in a specific format. Unlike excel which has a ‘cell space’ PowerBI uses a relational table structure; a series of tables connected together using matching columns. To add, PowerBI has an online service that controls data permissions, data refreshes, a notification system, and acts as a dataset server for pre-structured datasets - this service is where the users interact with data.

The Layers of PowerBI

PowerQuery Layer

Also known as the 'Transform Data' Section. This layer contains API connectors which connect to external or local data sources (extract). From the user's perspective this section or layer of the application is where you go to load in the data modify using 'M', the PowerQuery GUI or other languages like SQL, Python or R.

DAX Layer

The DAX layer is where you'll likely interact with PowerBI most of the time. This is where you define you table relationships, create visuals, and make measures. This layer is the most feature rich part of PowerBI and this is where PowerBI's performance is the most optimized.

Visualization Layer

This the layer where individual visual sit, and where you can customize them. Another option here is to use some of the pre-build aggregation functions like sum, count, average - however, it is always recommend for cleaniness you use user-defined dax calculations and functions.

PowerBI Service Layer

This is the component of PowerBI that sits on the cloud and it what really gives PowerBI it's distributive capabilities and seperates it from applications like Microsoft Excel. It allows for row-level permissions, and viewing of data by end users. This layer also has notifications and triggers which can be used, along with triggers to refresh data.

PowerBI in many ways is a number of interconnected Microsoft Applications that work together, and communicate with each other. The goal of this application is to show visual abstractions of data for business users (graphs, and tables); so the information can be interpreted and analyzed.

Now that you understand the different layers and structure of PowerBI we'll provide a notion of more components of those layers in PowerBI Desktop, as they are presented in the application.

Transform Data

1. API connectors that connect to external or local data sources (extract).

2. Transform or PowerQuery uses ‘M’, Python, SQL, and R.

3. Validation compiler - this validates the PowerBI data after the ETL has been performance and returns and error message if there are any conflicts or issues with the processing.

Main Screen

The main screen contains a menu like many other Microsoft applications at the top, along with a measure edit menu similar to MS Excel.

On the right are various layers, with the datasets, measures, columns being on the far right, with filters on the far left of this pane.

The main screen and tabs is the visualization area. On the far right you'll see 3 tabs in which you can look at the visualizations, data, or data schema. Recommend You toggle between these as you create measures and add Datasets

Each Visual

Each visual can be edited, and it's filters can be determine by many slicers. Click on the visual to edit, and use the GUI on the right to change features. This is a huge enchancement of Mircosoft's previous SSRS reports.

PowerBI allows users (dashboard creator) to ETL (extract load and transform) and creates Data Visualizations. PowerBI abstracts data in a relational table structure for the user (dashboard creator); a series of tables connected together using matching columns. PowerBI handles data in a columnar database structure, however, generates a relational structure for the dashboard creator. This design decision made by Microsoft makes the DAX language unintuitive unlike SQL whose syntax matches that of language.
